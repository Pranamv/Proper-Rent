# 13 — P1.6.2 Test Coverage Completion And Verification

P1.6.2 makes CI the authority for backend, frontend, e2e, contract, dependency, and production-smoke verification. Public API behavior is unchanged; the work adds coverage gates and drift checks around the existing Phase 1 surface.

## Acceptance Criteria

- Backend tests run with branch coverage and fail below 90% total coverage for `backend/app`.
- FastAPI OpenAPI output is checked into `contracts/openapi.json` and CI fails when it drifts.
- Frontend OpenAPI-derived TypeScript types are checked into `frontend/src/lib/api/openapi-types.ts` and CI fails when they drift.
- Frontend unit/component tests cover API client behavior, consent constants, admin redirect safety, renter/landlord forms, and chat widget success/failure paths.
- Playwright e2e covers public routes, renter intake, landlord intake, chat suggestion flow, and unauthenticated admin redirect against mocked API responses.
- Production smoke checks are read-only only: health, CORS, redirects, canonical metadata, robots/sitemap, security headers, admin login page, and no embedded local API URL.
- `https://www.properrent.co.uk` is the canonical public frontend host.

## Local Verification Bundle

Backend:

```bash
cd backend
ruff check .
ruff format --check .
mypy app
pytest
python scripts/export_openapi.py --check ../contracts/openapi.json
pip-audit -r requirements.txt
```

Frontend:

```bash
cd frontend
npm run lint
npm run typecheck
npm run test
npm run contract:check
npm run test:e2e
npm run build
npm run smoke:public
npm audit --audit-level=high
```

Read-only production smoke:

```bash
cd frontend
npm run smoke:prod
```

## Contract Workflow

Regenerate the checked artifacts whenever backend API schemas or routes intentionally change:

```bash
cd backend
python scripts/export_openapi.py --output ../contracts/openapi.json

cd ../frontend
npm run contract:types
```

Then run `python scripts/export_openapi.py --check ../contracts/openapi.json` and `npm run contract:check` before committing.

- The production smoke script accepts `PROD_SITE_ORIGIN`, `PROD_APEX_ORIGIN`, and `PROD_API_ORIGIN` overrides, but defaults to the deployed Proper Rent domains.
- If production DNS changes canonical host policy, update `frontend/src/lib/config.ts`, `docs/12-production-deployment.md`, and the smoke scripts together.
